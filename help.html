Hello and Welcome to the Decision Table Builder Tiny Tool!

As per GeeksforGeeks.org,

"A decision table is a brief visual representation for specifying which actions to perform depending on given conditions. The information represented in decision tables can also be represented as decision trees or in a programming language using if-then-else and switch-case statements.

A decision table is a good way to settle with different combination inputs with their corresponding outputs and is also called a cause-effect table. The reason to call cause-effect table is a related logical diagramming technique called cause-effect graphing that is basically used to obtain the decision table."



This Tiny Tool is used for building your own decision table.

The website has a point and click interface, so you can use your mouse or keyboard to operate it.

Changing Title of Table


The title can be entered in the "Title of Table" text box and after pressing the "Build Table" button, the table will be titled with user input.

Defining Number of Columns

The user can select the number of columns for each row in the decision table through the 'Number of Columns' dropdown menu.


Changing the number of columns will automatically update all rows within the builder, adding or removing the number of dropdown menus as specified.

Note: The maximum number of columns for the decision table is 10. The minimum number of columns is 1.

Adding New Actions or Conditions

The user can manually add up to 25 rows of actions and 25 rows of conditions.


When the green 'plus' icon is clicked, a corresponding row for action or condition will be generated.




Name of Row

The user can input the name of the row within the "Name of Row" text box.

The dropdown menus for each row allow the user to set column values to null '-', true 'T', or 'F'. All dropdown menus default to a null value when rows are created.

Building the Table

When satisfied with table parameters, the user can click the 'Build Table' button to see their table.

The table cannot be generated without a minimum of one action and one condition. The user can change information at their leisure and press the 'Build Table' button once more to see new changes.
